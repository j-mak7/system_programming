# Memory map

Программа имитирующая ядро операционной системы. Данное ядро осуществляет вывод отладочной информации на экран виртуальной машины и выводит основные параметры структуры memory map, передаваемой загрузчиком GRUB2 ядру операционной системы.
Примечания:
1. Разработка и отладка велась с использованием виртуальной машины QEMU;
2. Загрузчик GRUB2 сконфигурирован и настроен для запуска ядер операционных систем соответствующих спецификации Multiboot2;
3. Разрабатываемый образ операционной системы соответствует спецификации Multiboot2;
4. В качестве загрузчика QEMU используется UEFI-совместимый загрузчик OVMF;
5. Вывод информации осуществляется через кадровый буфер.

## Команды сборки

**Сборка ядра** выполняется командой `make`, "ядро" находится в файле **kernel.bin**.

#### Передача в виртуальную машину

Для передачи "ядра" в виртуальную машину, можно использовать виртуальный диск.
```bash
dd if=/dev/zero of=kernel.img bs=1024 count=102400
sudo mkfs -t vfat kernel.img
mkdir mnt_temp
sudo mount kernel.img mnt_temp
sudo cp kernel.bin mnt_temp
sudo umount mnt_temp
```
#### Запуск QEMU
```bash
./qemu-system-x86_64 -drive if=pflash,format=raw,readonly=on,file=/usr/share/OVMF/OVMF_CODE_4M.fd -smp 2,sockets=1,cores=2,threads=1 -m 4G -serial stdio -hda disk.qcow2 -net nic -net user -cdrom /home/julia/lab3_sp/kernel.img
```
#### Запуск ядра

В меню GRUB необходимо нажать клавишу `c`, чтобы открылась консоль GRUB.

```
multiboot2 (cd0)/kernel.bin
boot
```

После этого, экран должен стать синим, а на нём появится информация о карте памяти.
