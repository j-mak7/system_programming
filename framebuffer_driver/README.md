# Framebuffer driver
Задание: спроектировать и реализовать на языке C драйвер PCIe-устройства для ОС Linux.
**Примечания**:
1. Разработка и отладка ведётся с использованием виртуальной машины QEMU;
2. Для разработки, отладки и демонстрации, в качестве памяти виртуального PCIe-устройства используется внешний бинарный файл;
3. Необходимо предусмотреть возможность конфигурирования через sysfs. 
## Описание
Драйвер при записи в него, принимает поток байт, который представляет собой изображение в формате BMP и отображает его в кадровый буфер.
Если изображение меньше "экрана", то его необходимо отобразить в левом верхнем углу, не перерисовывая остаток изображения. Если изображение больше экрана, изображение обрезается по размер буфера.
При чтении - драйвер выводит кадровый буфер в виде потока байтов представляющих собой изображение в формате BMP.
#### Через ioctl и sysfs:
- Считываются поддерживаемые режимы
- Считывается текущий режим
- Настраивается текущий режим
- Очистка экрана
#### Управляющие регистры
- **RO** Количество режимов вывода (разрешение экрана, например 800х600, 1024х786 и т.д.)
- **RO** Регистр с адресом структуры, содержащей режимы (64 бита)
- **RO** Регистр с адресом буфера (64 бита)
- **RO** Регистр статуса (готов к записи, ошибка)
- **RO** Буфер с режимами (количество режимов из регистра умножить на размер структуры)
- **RW** Кадровый буфер (равен количеству точек на «экране», если режим работы 800х600, то размер 800х600x3, 3 так как этого достаточно для задания цвета в формате RGB)
### Возможности
С гостевой ОС изображение в формате **bmp** записывается в *bar2.bin*, откуда его считывает программа на хосте, которая выполняет роль экрана. Также возможно обратное действие, когда происходит передача изображения с хоста на гостевую ОС.

## Команды сборки

#### Запуск QEMU
```bash
./qemu-system-x86_64 -drive if=pflash,format=raw,readonly=on,file=/usr/share/OVMF/OVMF_CODE_4M.fd -smp 2,sockets=1,cores=2,threads=1 -m 4G -vga none -serial stdio -hda disk.qcow2 -net nic -net user -object memory-backend-file,size=2M,share=on,mem-path=bar2.bin,id=membar2 -device pci-testdev,membar=2M,memdev=membar2 -cdrom debian-live-12.10.0-amd64-standard.iso -machine pc,accel=tcg -virtfs local,path=/home/julia/driver,mount_tag=host_driver,security_model=mapped
```
#### Сборка и подключение драйвера на гостевой ОС
```bash
make && sudo insmod mydriver.ko
```
#### Тестирование работы драйвера на запись изображения с гостевой ОС на хост
```bash
echo 2 > /sys/class/mydriver/mydriver/count_mode
echo "800*600 1*1" > /sys/class/mydriver/mydriver/buf_mode
echo 0 > /sys/class/mydriver/mydriver/current_mode
cat driver/image.bmp > /dev/mydriver
```

#### Запуск программы на хосте
```bash
 gcc -o h_app host_application.c && ./h_app
```

В данном примере:
1. Хост - 20 CPU, 16 Gb ОЗУ, ОС Windows 11
2. Виртуальная машина (ВМ) - 1 vCPU, 4 Gb ОЗУ, ОС Debian 12
