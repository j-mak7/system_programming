# UEFI Multiboot2 Загрузчик

Приложение исполняемое в UEFI и реализующее функционал загрузчика соответствующего спецификации multiboot2. Формирует две загрузочные структуры (кадровый буфер и карту памяти), находит ядро операционной системы в файловой системе, проверяет его на наличие корректного заголовка multiboot2, завершает стадию загрузки UEFI (если это требуется в заголовке ядра) и передает управление ядру ОС.
Примечания:
1. Разработка и отладка велась с использованием виртуальной машины QEMU;
2. В качестве загрузчика QEMU использовался UEFI-совместимый загрузчик OVMF;
3. Разработка и отладка велась на основе проекта с открытым исходным кодом EDK2;
4. Загрузчик принимает путь до файла ядра как аргумент командной строки;
5. Загрузчик проверяет все записи в заголовке ядра в соответствие со спецификацией Multiboot2;
6. Загрузчик заполняет таблицу с загрузочными записями в соответствие со спецификацией
multiboot2;
7. Вывод информации осуществляется через кадровый буфер.

## Команды сборки

### Разворачивание EDK2

Требуется установить следующее дополнительное ПО на свою ОС:

1. build-essential
2. uuid-dev
3. nasm
4. acpica-tools
5. python3
6. GCC

Далее загрузить репозиторий edk2 и проинициализировать подмодули:

```bash
git clone https://github.com/tianocore/edk2.git
cd edk2
git checkout edk2-stable202511
git submodule update --init
```

Хотя бы один раз собрать проект Ovmf

```bash
./OvmfPkg/build.sh
```

### Сборка тестового приложения

Скопируйте папку с тестовым приложение в директорию ` MdeModulePkg/Application`.

В файл `MdeModulePkg/MdeModulePkg.dsc`, в секцию [Components] добавьте путь до конфигурационного файла тестового приложения:

```
[Components]
  ...
  MdeModulePkg/Application/BootLoader/BootLoader.inf
  ...
```

Запустите сборку отдельного драйвера:

```bash
build -a X64 -t GCC5 -p MdeModulePkg/MdeModulePkg.dsc -m MdeModulePkg/Application/BootLoader/BootLoader.inf -b DEBUG
```

### Запуск тестового приложения

Скопируйте в корень образа диска `uefi.img` тестовое приложение и тестовое ядро.

Запустите QEMU

```bash
qemu-system-x86_64 -drive if=pflash,format=raw,readonly=on,file=/usr/share/OVMF/OVMF_CODE_4M.fd -m 4G -serial stdio -vga std -hda disk.qcow2 -net nic -net user -cdrom uefi.img
```

Сразу после появления логотипа UEFI нажмите F2 для загрузки меню настройки UEFI.
В меню перейдите в **Boot Manager**  из запустите **EFI Internal Shell**.
Перейдите в файловую систему виртуального диска `uefi.img`. 
Запустите BootLoader.efi

```
FS1:\>BootLoader.efi
```

Загрузчик найдет `kernel.bin`, проверит его заголовок и передаст управление. Экран станет синим с информацией о карте памяти.
